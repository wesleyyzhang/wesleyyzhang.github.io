CS 180 Final Project: Image Quilting

In this project, I aim to implement the image quilting algorithm for texture synthesis and transfer. We attempt to generate new texture from an already existing sample image. The main idea is to sample patches and overlay the samples over already existing portions of the image that have overlapping patterns. We will use many different techniques to generate the new texture and compare the results to each other. The test image we will be using is shown below:

Part 1: Randomly Sampled Texture

The first approach we will take is to simply randomly sample patches from the test image and add them to the output image until we have filled up all of the space in the output image. Part 2: Overlapping Patches

In this part, we will use quilting to more effectively generate new texture. We will implement the quilt_simple(sample, out_size, patch_size, overlap, tol) function, which creates an output image by randomly sampling and stiching square patches from the test image. We start from the upper-left corner and proceed row-by-row. Patches will overlap by patch_size pixels in the vertical direction and by overlap pixels in the horizontal direction. For each new patch samples, we will calculate the SSD between the overlapping regions of existing patches. The SSD will be calculated using the following equation: ssd_cost = ((M*T)**2).sum() - 2 * cv2.filter2D(I, ddepth=-1, kernel = M*T) + cv2.filter2D(I ** 2, ddepth=-1, kernel=M). Afterwards, we will randomly select one of the tol lowest-cost patches to place in the output image. We can see that the results below are much better than the previous part.

Part 3: Seam Finding

In this next part, we will incorporate seam finding to remove edge artifacts from the overlapping patches. The quilt_cut function will enhance the quilting algorithm by finding minimum-cost paths through overlapping regions regions between patches. The seam cost is calculated as the squared RGB differences between the output image and the new patch. The vertical seams can be found by transposing the cost matrix before and after applying the cut function: cut(bndcost.T).T. We can again see that it is much more effective than randomly sampling patches.

Part 4: Texture Transfer

Lastly, we will create a function texture_transfer based on the quilt_cut function for creating a texture sample that is guided by a pair of sample/target correspondence images. There is an additional cost term based on the difference between the sampled source patch and the target patch at the location to be filled. This method seems very effective at generating new texture, as we can see that the gaps between the bricks are effectively aligned. The bricks seem more aligned than in Part 3.

Bells and Whistles: Iterative Texture Transfer

We will now implement iterative texture transfer according to the research paper by Efros and Freeman. In this algorithm, we initialize an alpha value and gradually decrease this value in subsequent iterations. We also use a weighted combination of the original target image and the previous iteration's result as guidance. This will allow us to maintain the target's features while progressively improving the quality of the image.
